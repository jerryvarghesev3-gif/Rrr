Description
Create the basic GUI failure mode and error scenario layout to handle different failure conditions in the application workflow. This task focuses only on preparing the UI structure, placeholders, and screen flow for error handling. The screen should be ready to display failure title, status, error message, possible reason, and next-step guidance once backend wiring is completed in future sprints.

Acceptance Criteria
	•	Failure mode screen layout is created and accessible from the GUI flow.
	•	Screen contains placeholders for error title, status, detailed message, and user guidance.
	•	Different failure scenarios can be visually represented through the same layout structure.
	•	UI clearly distinguishes failure state from normal and success states.
	•	Layout is reusable for future backend-driven error data binding.
	•	Screen design is consistent with the overall launcher GUI style.
	•	No backend integration is required in this task; only static/basic UI setup is completed.

Output
GUI failure mode and error scenario layout implemented, ready to display backend-driven error details, status, and user guidance in future integration.
