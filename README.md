Description

Create the GUI screens for failure mode, error handling, and CAN/SFTP status using QT/QML.
This task focuses on building the UI layouts and visual components required to represent error states and connection status, without implementing backend logic.

The screens will include placeholders for error messages, status indicators, and recovery actions, and will be designed to integrate with backend logic in future sprints.

⸻

✅ Acceptance Criteria
	•	Failure mode screen UI is created in QT/QML
	•	Error handling screen displays placeholder messages and status indicators
	•	CAN/SFTP status section is implemented with visual placeholders (e.g., connected/disconnected states)
	•	UI components (labels, icons, buttons) are properly aligned and visible
	•	Screens support navigation from main or relevant sections
	•	No backend integration is implemented (UI-only placeholders)
	•	Layout supports future dynamic updates (status, errors, retry actions)
	•	No UI crashes or rendering issues during navigation

⸻

📦 Output
	•	Failure mode screen UI implemented
	•	Error handling screen with placeholder messaging
	•	CAN/SFTP status screen with visual indicators
	•	Structured UI ready for backend-driven status updates
	•	Stable and extensible GUI components prepared for integration
