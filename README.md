Description

Implement the navigation layout and create the connection screen UI in QT/QML as part of the launcher GUI.
This task focuses on defining the structural layout, navigation flow, and basic UI components required for connection-related interactions (e.g., CAN-USB, SFTP), without integrating backend logic.

The UI will act as a placeholder and foundation for future backend wiring, where actual connection handling and data display will be implemented in later sprints.

⸻

✅ Acceptance Criteria
	•	Navigation layout is implemented using QT/QML
	•	User can navigate between main page and connection screen
	•	Connection screen UI is created with required placeholders (e.g., CAN, SFTP sections)
	•	UI elements (buttons, input fields, labels) are visible and properly aligned
	•	Layout follows defined design/structure for future integration
	•	No backend calls are triggered (UI-only implementation)
	•	Navigation transitions work without UI glitches or crashes
	•	Screens are ready for future data binding and backend integration

⸻

📦 Output
	•	Navigation layout structure implemented in QT/QML
	•	Connection screen UI created with placeholders for CAN/SFTP
	•	Basic UI navigation flow established
	•	UI foundation prepared for backend connectivity in future sprints
	•	Clean, stable, and extensible GUI layout ready for integration
